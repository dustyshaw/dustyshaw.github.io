There seemed to be so many differences in how you render and comput components. Why React Native?

React Native stems from one of the most popular JavaScript front-end libraries: React js. While React js is a front-end component library built off of JavaScript, React Native is just for creating native apps (the kind of apps you find in the app store on iOS, Android, and Windows). Confusing right!?

After some research into different frameworks for mobile development, I found that React Native has some benefits:

  • Components can be reaused for web based applications written in React js (awesome! Reduced duplication)
  • Write platform specific code with just JavaScript (great! Don't have to learn Java or that pesky Swift to make mobile apps)
  • Seems faster than other front-end frameworks like Angular

At the end of the day, the programming language you choose might not make as much of a difference than you think, so I try not to stress this part too much.

Planning my Practice Project

To begin, I decided to pencil in some designs of what I want my app to look like. This helps me understand the different functions that my app requires, and it gives me a great jumping off point.

Now that I had the general layout of what I wanted the app to look like, it was time to develop some requirements.

Users should be able to...

  1. Add a food item to a specific day.
  2. See an overview of how many calories they have eaten per day.
  3. See the macros of what they have eaten for that day (protein, carbs, fats)
  4. Scan a food item to find it via a food API

With these requirements, I now know what the minimum viable product is.
